# GitHub Pages 发布流程

当前最新版网站目录：

`/Users/hawksky/Desktop/modern phi/modern philosophy website`

当前 GitHub Pages 仓库：

`https://github.com/hawks0261-sudo/modern-philosophy-website.git`

## 手动发布 3 步版

1. 进入网站目录

```bash
cd "/Users/hawksky/Desktop/modern phi/modern philosophy website"
```

2. 提交本次修改

```bash
git add -A
git commit -m "Update site"
```

3. 推送到 GitHub Pages

```bash
git push origin main
```

推送完成后，通常等待几分钟即可在下面地址看到更新：

`https://hawks0261-sudo.github.io/modern-philosophy-website/`

## 自动版：以后正式上线时直接发给 Codex 的命令

下面这些不是终端命令，而是你以后可以直接发给我的指令模板。到时你准备好了，直接复制一句给我即可。

### 1. 正式上线前的最终检查

```txt
请按正式上线流程检查当前网站，不要直接发布，先帮我检查页面、链接、图片、GitHub Pages 发布状态，并列出最后确认项。
```

### 2. 正式上线并发布 GitHub Pages

```txt
我已确认正式上线，请在 "/Users/hawksky/Desktop/modern phi/modern philosophy website" 执行最终发布，提交并推送到 GitHub Pages。
```

### 3. 正式上线并绑定自定义域名

```txt
我已购买域名，请帮我把当前网站正式上线到 GitHub Pages，并告诉我 CNAME 内容和域名 DNS 应该怎么配置。域名是：example.com
```

把 `example.com` 改成你实际购买的域名即可。

### 4. 正式上线后继续更新

```txt
请把当前网站作为正式线上版本处理，检查本次改动后直接发布到 GitHub Pages。
```

## 重要说明

- GitHub Pages 的服务器在 GitHub，不在你的本地电脑上。
- 你本地电脑只负责编辑、提交、推送。
- 你电脑关机后，网站仍然可以正常访问。
- 购买域名后，仍然可以继续使用 GitHub Pages 托管，不需要把网站迁移到你自己的本地服务器。
